# webs-mk-journal

## 项目简介

`webs-mk-journal` 是一个用于归档和发布独立 HTML 页面的仓库。这里存放着多个不针对特定域名或网站的静态页面，旨在以简洁的方式保存网页快照、笔记或小型工具页面，方便日后查阅与分享。

本仓库不依赖任何后端服务，所有内容均为纯前端静态页面，可直接通过 GitHub Pages 或其他静态托管服务访问。

## 目录结构

```
.
├── README.md          # 本说明文件
├── index.html         # 仓库入口页（可选）
├── pages/             # 归档页面存放目录
│   ├── example1/      # 示例归档页面文件夹
│   │   └── index.html
│   └── example2/
│       └── index.html
└── assets/            # 公共资源（CSS、JS、图片等）
    └── style.css
```

- `pages/`：存放所有独立 HTML 页面的子目录，每个子目录代表一个归档项目。
- `assets/`：存放共用的样式、脚本或图片资源，供各页面引用。

## 页面归档说明

每个归档页面应独立存放在 `pages/` 下的一个子目录中，并包含完整的 HTML 文件。建议遵循以下规范：

- 页面应自包含，避免依赖外部资源（除了仓库内的 `assets/` 目录）。
- 如引用外部资源（如字体、库），请尽量使用相对路径或确保资源长期可用。
- 可在页面内附上简短的归档说明或来源备注，便于理解上下文。

示例页面结构：

```
pages/my-page/
├── index.html
├── style.css
└── script.js
```

## 维护说明

- 新增页面：在 `pages/` 中创建新文件夹，放入 HTML 文件及其依赖资源，并更新 `index.html`（如有）中的导航链接。
- 更新页面：直接修改对应文件夹内的文件，保持版本记录可通过 Git 提交信息追溯。
- 删除页面：移除对应文件夹，并清理入口页的链接。
- 贡献者请确保提交的页面内容合法、不侵犯他人权益，并尽量保持轻量简洁。
